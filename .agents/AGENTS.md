# Project Rules

- **Ask When Unsure**: Ask the user questions whenever you are unsure of requirements, design decisions, or implementation paths.
- **Consult Documentation**: Always check the repository documentation (e.g., `docs/`, `AGENT.md`, `CONTEXT.md`) for answers before asking or making assumptions.
- **Address Contradictions**: If you cannot find an answer in the docs or encounter a contradiction, ask the user for clarification.
- **Commit Approvals**: Always ask the user for explicit approval before making any git commits.
- **Simplicity & Readability**: Focus on simple, clear, readable code. Avoid adding unnecessary lines or complexity.
- **Refactoring Strategy**: When refactoring, focus on removing rather than adding code, keeping it clean and minimal.
