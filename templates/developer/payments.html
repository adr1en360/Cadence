API Reference

Payments API

The Payments API allows you to inspect payment histories and trigger payment operations such as refunds for your subscription renewals.

Refund Payment

POST /api/payments/{payment_id}/refund

Triggers a refund request for a successful subscription payment. This communicates with Nomba's API under the hood to return the billed funds to the subscriber's bank card. Note that only successful (succeeded) payments can be refunded.

Path Parameter Type Required Description
payment_id string Yes The unique identifier of the payment to be refunded.
Response JSON (200 OK)
{ "message": "Payment refunded successfully", "status": "refunded" }
Error JSON (400 Bad Request)
{ "detail": "Only succeeded payments can be refunded. Current status: refunded" }

Integration Code Examples

cURL
curl -X POST "https://cadence.onrender.com/api/payments/pay_9a2f6b3e-7c1d-4c8a-8e2f-5d6c7b8a9c0d/refund" \ -H "Authorization: Bearer YOUR_API_KEY"
Python (requests)
import requests payment_id = "pay_9a2f6b3e-7c1d-4c8a-8e2f-5d6c7b8a9c0d" url = f"https://cadence.onrender.com/api/payments/{payment_id}/refund" headers = { "Authorization": "Bearer YOUR_API_KEY", "Content-Type": "application/json" } response = requests.post(url, headers=headers) print(response.json())
Node.js (fetch)
const apiKey = "YOUR_API_KEY"; const paymentId = "pay_9a2f6b3e-7c1d-4c8a-8e2f-5d6c7b8a9c0d"; fetch(`https://cadence.onrender.com/api/payments/${paymentId}/refund`, { method: "POST", headers: { "Authorization": `Bearer ${apiKey}`, "Content-Type": "application/json" } }) .then(res => res.json()) .then(data => console.log(data)) .catch(err => console.error(err));
