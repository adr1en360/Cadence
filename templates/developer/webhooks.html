Webhooks

Webhook Integration

Webhooks allow Cadence to notify your application server in real-time when billing events happen, such as successful renewals, subscription cancellations, or payment failures.

Specify your destination webhook endpoint in your Merchant Dashboard. Cadence sends HTTP POST requests with a JSON body and an authentication signature header.

Outbound Signature Verification

Every webhook body sent by Cadence is signed using your merchant account's private Webhook Signing Secret. The signature is transmitted in the X-Cadence-Signature header.

To verify the payload is genuine, parse the JSON body and compute the HMAC-SHA256 hash of its compact JSON representation using your Webhook Secret.

Python
import hmac import hashlib import json expected_sig = request.headers.get("X-Cadence-Signature") # Reconstruct the exact compact body layout signed by Cadence payload_dict = request.json() compact_body = json.dumps(payload_dict, separators=(',', ':')) computed_sig = hmac.new( key=WEBHOOK_SECRET.encode("utf-8"), msg=compact_body.encode("utf-8"), digestmod=hashlib.sha256 ).hexdigest() if hmac.compare_digest(computed_sig, expected_sig): # Payload is verified and safe process_event()
Node.js
const crypto = require('crypto'); const expectedSig = req.headers['x-cadence-signature']; const compactBody = JSON.stringify(req.body); const computedSig = crypto .createHmac('sha256', WEBHOOK_SECRET) .update(compactBody) .digest('hex'); if (crypto.timingSafeEqual(Buffer.from(computedSig), Buffer.from(expectedSig))) { // Payload is verified and safe processEvent(); }
Go
import ( "crypto/hmac" "crypto/sha256" "encoding/hex" "encoding/json" ) func VerifyCadenceWebhook(bodyBytes []byte, secret string, expectedSig string) bool { var raw interface{} json.Unmarshal(bodyBytes, &raw) compactBody, _ := json.Marshal(raw) mac := hmac.New(sha256.New, []byte(secret)) mac.Write(compactBody) computedSig := hex.EncodeToString(mac.Sum(nil)) return hmac.Equal([]byte(computedSig), []byte(expectedSig)) }

Event Types & Payloads

1. subscription.status_updated

Fired when a subscriber's state machine changes (e.g. from active to past_due or when a subscription enters a grace period).

Payload Example
{ "event_type": "subscription.status_updated", "created_at": "2026-07-01T12:05:00Z", "data": { "subscription_id": "sub_48f61c20-7dbe-4...", "old_state": "active", "new_state": "past_due", "customer_email": "customer@example.com" } }

2. subscription.cancelled

Fired when a subscriber is cancelled (either immediately or scheduled to be cancelled at the end of the billing period).

Payload Example
{ "event_type": "subscription.cancelled", "created_at": "2026-07-02T15:20:00Z", "data": { "subscription_id": "sub_48f61c20-7dbe-4...", "customer_email": "customer@example.com", "cancel_at_period_end": true } }

3. payment.succeeded

Fired when a recurring subscription renewal payment succeeds via Nomba card charging.

Payload Example
{ "event_type": "payment.succeeded", "created_at": "2026-07-01T12:00:05Z", "data": { "payment_id": "pay_9a2f6b3e-7c1d-4c8a-8e2f-5d6c7b8a9c0d", "subscription_id": "sub_48f61c20-7dbe-4...", "amount": 2000.00, "customer_email": "customer@example.com" } }

4. payment.failed

Fired when a recurring renewal charge fails. This initiates the 11-day retry dunning cycle.

Payload Example
{ "event_type": "payment.failed", "created_at": "2026-07-01T12:00:00Z", "data": { "subscription_id": "sub_48f61c20-7dbe-4...", "amount": 2000.00, "customer_email": "customer@example.com", "reason": "Insufficient funds" } }

5. payment.refunded

Fired when a previous successful payment is refunded to the customer's card.

Payload Example
{ "event_type": "payment.refunded", "created_at": "2026-07-02T10:15:30Z", "data": { "payment_id": "pay_9a2f6b3e-7c1d-4c8a-8e2f-5d6c7b8a9c0d", "nomba_transaction_id": "txn_8c1b2a3d...", "amount": 2000.00, "customer_email": "customer@example.com" } }

Sandbox Webhook Simulation

To test your webhook endpoint's logic without needing actual bank card operations, you can trigger simulated Nomba webhook events using the following sandbox test endpoints.

Simulate Payment Success

POST /webhooks/test-success
Request Body Example
{ "order_ref": "order_ref_abc123xyz", "transaction_id": "test_txn_12345", "token_key": "test_token_key_abcde" }

Simulate Payment Failure

POST /webhooks/test-failed
Request Body Example
{ "order_ref": "order_ref_abc123xyz" }
