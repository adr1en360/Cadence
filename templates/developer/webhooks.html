Webhooks

Webhook Integration

Webhooks allow Cadence to notify your application server in real-time when billing events happen, such as successful renewals, subscription cancellations, or payment failures.

Specify your destination webhook endpoint in your Merchant Dashboard. Cadence sends HTTP POST requests with a JSON body and an authentication signature header.

Outbound Signature Verification

Every webhook body sent by Cadence is signed using your merchant account's private Webhook Signing Secret. The signature is transmitted in the X-Cadence-Signature header.

To verify the payload is genuine, parse the JSON body and compute the HMAC-SHA256 hash of its compact JSON representation using your Webhook Secret:

Python HMAC Verification Example
import hmac import hashlib import json expected_sig = request.headers.get("X-Cadence-Signature") # Reconstruct the exact compact body layout signed by Cadence payload_dict = request.json() # Parse request body as JSON dict compact_body = json.dumps(payload_dict, separators=(',', ':')) computed_sig = hmac.new( key=WEBHOOK_SECRET.encode("utf-8"), msg=compact_body.encode("utf-8"), digestmod=hashlib.sha256 ).hexdigest() if hmac.compare_digest(computed_sig, expected_sig): # Payload is verified and safe process_event()

Event Types & Payloads

1. subscription.status_updated

Fired when a subscriber's state machine changes (e.g. from active to past_due or when a subscription is cancelled).

Payload Example
{ "event_type": "subscription.status_updated", "created_at": "2026-07-01T12:05:00Z", "data": { "subscription_id": "sub_48f61c20-7dbe-4...", "old_state": "active", "new_state": "past_due", "customer_email": "customer@example.com" } }
