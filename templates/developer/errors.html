Errors

Error Codes Reference

Cadence uses standard HTTP response status codes to indicate the success or failure of API requests. Responses in the 4xx range denote user errors, while 5xx range responses indicate internal server anomalies.

Standard Error Responses

HTTP Status Code / Detail Description / Remedy
401 Missing API Key / Invalid Credentials Authorization header is absent, uses incorrect prefix structure, or the key secret hash does not match DB values.
404 Plan not found / Subscription not found The requested UUID does not exist, or is scoped to a different merchant key credential.
400 Invalid subscription state transition Requested state modification is illegal (e.g. attempting to transition an expired subscription back to active).
500 Failed to initialize subscription checkout Could not retrieve token credentials or checkout link from Nomba sandbox APIs due to network timeout or downstream account suspension.
