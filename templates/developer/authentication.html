Security

Authentication

All requests made to Cadence developer API endpoints must be authenticated. Cadence uses Bearer API tokens generated from your Merchant Dashboard portal.

Include your secret API key in the standard Authorization header of every outgoing request:

HTTP Request Header
Authorization: Bearer cd_dev_26284f69c7ef95a...

Key Scopes & Best Practices

  • Keep Secrets Secure: Since Cadence API keys grant access to billing transactions, never expose keys in client-side code (web browsers, mobile applications, etc.). Always route operations through your secure server backend.
  • Key Prefixing: Cadence keys use prefixes (e.g. cd_dev_ or cd_live_) that allow local parsers to check structure without calling the database.
  • Scoping: Every resource created (plans, subscriber details, payment histories) is scoped to the owner of the key, ensuring isolation of sandbox environments.
